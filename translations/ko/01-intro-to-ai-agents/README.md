# AI 에이전트와 활용 사례 소개

"초보자를 위한 AI 에이전트" 과정에 온 걸 환영해요! 이 과정에서는 AI 에이전트를 만드는 데 필요한 기본 지식과 실제 예제를 배울 수 있어요.

다른 학습자들, AI 에이전트 개발자들과 만나서 질문도 하고 교류할 수 있어요.

이번 강의에서는 AI 에이전트가 뭔지, 그리고 우리가 만드는 앱이나 워크플로에서 어떻게 활용할 수 있는지 알아볼게요.

## 이번 강의 내용

- AI 에이전트란? 어떤 종류가 있을까?
- AI 에이전트에 적합한 활용 사례는?
- 에이전트 솔루션 설계 시 기본 구성 요소는?

- 사용자와 고객 모두를 위한 에이전트 솔루션을 설계할 수 있어요

## AI 에이전트 정의 및 종류

### AI 에이전트란?

AI 에이전트는 **LLM(대규모 언어 모델)**이 **도구와 지식**에 접근해서 **작업을 수행**할 수 있게 해주는 **시스템**이에요.

좀 더 자세히 살펴볼게요:

| 구성 요소 | 설명 | 예시 (여행 예약 에이전트) |
|-----------|------|---------------------------|
| **환경** | 에이전트가 작동하는 공간 | 여행 예약 시스템 |
| **센서** | 환경에서 정보를 수집 | 호텔 가용성, 항공편 가격 |
| **액추에이터** | 환경을 변경하는 행동 수행 | 객실 예약하기 |

![AI 에이전트란 무엇인가?](../../../translated_images/what-are-ai-agents.1ec8c4d548af601a3a78c6c02e5c355d19c06a4a74fe93e3609a1d08e8c15689.ko.png)

**LLM의 역할**: 에이전트 개념은 LLM 이전에도 있었지만, LLM을 쓰면 인간의 언어와 데이터를 해석할 수 있어서 환경 정보를 이해하고 계획을 세우는 게 가능해져요.

**작업 수행**: 일반 LLM은 프롬프트에 따라 콘텐츠나 정보만 생성하지만, AI 에이전트 안에서는 사용자 요청을 해석하고 도구를 써서 실제 작업을 수행할 수 있어요.

**도구 접근**: 에이전트가 쓸 수 있는 도구는 ① 환경에서 제공하는 것과 ② 개발자가 정의한 것으로 결정돼요.

**메모리+지식**: 단기 메모리(대화 맥락)와 장기 지식(다른 시스템에서 가져온 정보)을 활용할 수 있어요. 예를 들어 고객 DB의 여행 선호 정보 같은 거죠.

### 에이전트 유형

| 유형 | 특징 | 예시 |
|------|------|------|
| **단순 반사 에이전트** | 규칙에 따라 즉시 반응 | 이메일 내용 해석해서 불만 사항을 고객 서비스로 전달 |
| **모델 기반 반사 에이전트** | 세계 모델을 기반으로 판단 | 과거 가격 데이터로 변동이 큰 경로 우선 추천 |
| **목표 기반 에이전트** | 목표 달성을 위한 계획 수립 | 출발지→목적지 최적 경로(차/대중교통/비행기) 결정 |
| **유틸리티 기반 에이전트** | 선호도와 트레이드오프 고려 | 편리함 vs 비용 비교해서 최적 옵션 제안 |
| **학습 에이전트** | 피드백으로 시간이 지나며 개선 | 여행 후 설문 피드백으로 다음 추천 개선 |
| **계층적 에이전트** | 상위 에이전트가 하위에 작업 분배 | 예약 취소를 세부 작업으로 나눠서 처리 |
| **다중 에이전트 시스템** | 여러 에이전트가 협력/경쟁 | 호텔·항공·엔터테인먼트 에이전트가 각각 담당 |

## AI 에이전트를 언제 쓰면 좋을까?

여행 예약 사례로 여러 에이전트 유형을 설명했는데요, 이 애플리케이션은 과정 전체에서 계속 사용할 거예요.

AI 에이전트가 가장 잘 맞는 경우:

![AI 에이전트를 사용할 때?](../../../translated_images/when-to-use-ai-agents.54becb3bed74a479f5caca9c951132ce81d482a6704bcd22e5a600dbabc9434e.ko.png)

| 상황 | 설명 |
|------|------|
| **개방형 문제** | 워크플로에 하드코딩할 수 없고, LLM이 필요한 단계를 결정해야 할 때 |
| **다단계 프로세스** | 여러 턴에 걸쳐 도구나 정보를 사용해야 하는 복잡한 작업 |
| **시간에 따른 개선** | 피드백을 받아 점점 더 나은 결과를 내야 할 때 |

> 💡 AI 에이전트 사용 시 추가 고려사항은 "신뢰할 수 있는 AI 에이전트 구축" 강의에서 다뤄요.

## 에이전트 솔루션의 기본

### 에이전트 개발

AI 에이전트 시스템 설계의 첫 단계는 도구, 작업, 행동을 정의하는 거예요. 이 과정에서는 **Azure AI Agent Service**를 사용해요.

제공 기능:
- OpenAI, Mistral, Llama 같은 오픈 모델 선택
- Tripadvisor 같은 제공자의 라이선스 데이터 사용
- 표준화된 OpenAPI 3.0 도구 사용

### 에이전트 패턴

LLM과의 소통은 프롬프트로 이루어지는데, AI 에이전트는 반자율적이라 환경 변화마다 수동으로 프롬프트할 수 없어요. 그래서 **에이전트 패턴**을 사용해 여러 단계에 걸쳐 확장 가능하게 프롬프트해요.

이 과정에서 인기 있는 에이전트 패턴 몇 가지를 배울 거예요.

### 에이전트 프레임워크

에이전트 프레임워크는 개발자가 코드로 에이전트 패턴을 구현할 수 있게 해줘요. 템플릿, 플러그인, 도구를 제공해서 에이전트 간 협업도 쉬워지고, 시스템의 관찰 가능성과 디버깅도 개선돼요.

이 과정에서는:
- **AutoGen** - 연구 중심 프레임워크
- **Semantic Kernel** - 프로덕션 준비된 Agent 프레임워크

를 탐구할 거예요.
